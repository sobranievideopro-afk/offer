# Resume Bot — Telegram-бот для анализа резюме

MVP со следующими функциями:
- 📄 Анализ резюме + оценка 0-100 (`/analyze`)
- ✍️ Улучшение формулировок (`/improve`)
- 🎯 Адаптация под вакансию (`/tailor`)
- 🎤 Интервью-тренажёр с фидбеком (`/interview`)
- 📸 Деловое фото — заглушка, нужен отдельный image-API (`/photo`)

## Быстрый запуск локально (для теста)

1. Установи Node.js 18+ (https://nodejs.org)
2. Получи токен бота у **@BotFather** в Telegram (команда `/newbot`)
3. Получи API-ключ Anthropic на https://console.anthropic.com
4. В папке проекта:

```bash
npm install
cp .env.example .env
```

5. Открой `.env` и вставь свои ключи:

```
BOT_TOKEN=твой_токен_от_botfather
ANTHROPIC_API_KEY=твой_ключ_anthropic
CLAUDE_MODEL=claude-sonnet-5
```

6. Запусти:

```bash
npm start
```

7. Открой своего бота в Telegram и напиши `/start`

## Как протестировать сценарий

1. Пришли файл резюме (.pdf, .docx или .txt)
2. `/analyze` — получишь оценку 0-100
3. `/improve` — улучшенная версия резюме
4. Пришли текст вакансии обычным сообщением (если не присылал раньше)
5. `/tailor` — адаптация под вакансию
6. `/interview` — начнётся тренировка: бот задаст 6 вопросов, после каждого ответа даст фидбек, в конце — итоговый отчёт

## Деплой на Railway (чтобы бот работал 24/7)

1. Залей код в GitHub-репозиторий
2. На https://railway.app: New Project → Deploy from GitHub repo
3. В настройках проекта (Variables) добавь:
   - `BOT_TOKEN`
   - `ANTHROPIC_API_KEY`
   - `CLAUDE_MODEL`
4. Railway автоматически определит `npm start` из package.json и запустит бота
5. Bot работает в режиме polling — отдельный домен/webhook не нужен

## Подключение базы данных (Postgres) на Railway

Позволяет хранить историю пользователей, анализов резюме и ответов на интервью — не только смотреть логи глазами, но и делать выборки.

### Шаг 1: Добавь Postgres к проекту

1. Открой свой проект на Railway
2. Нажми **+ New** → **Database** → **Add PostgreSQL**
3. Railway сам создаст базу и автоматически пропишет переменную `DATABASE_URL` в твоём проекте — вручную ничего вводить не нужно

### Шаг 2: Обнови код (см. секцию "Что нового" ниже) и запушь в GitHub

Railway передеплоит бота. При старте бот сам создаст все нужные таблицы (`users`, `events`, `resume_analyses`, `interview_answers`) — миграции запускать не нужно.

### Шаг 3: Проверь, что база работает

В логах Railway при запуске должна появиться строка:
```
✅ Схема базы данных проверена/создана
```

### Шаг 4: Смотри данные

В Railway у плагина Postgres есть вкладка **Data** со встроенным SQL-редактором. Готовые примеры запросов — в файле `useful_queries.sql` в этом проекте (кто пользователи, средний score, воронка, самые активные и т.д.).

## Что нового (файлы для обновления в GitHub)

- `package.json` — добавлена зависимость `pg`
- `src/db.js` — **новый файл**, подключение к Postgres и функции сохранения данных
- `src/bot.js` — добавлен middleware для сохранения пользователя + вызовы записи событий рядом с каждым действием
- `index.js` — при старте создаёт таблицы в БД перед запуском бота
- `.env.example` — добавлена переменная `DATABASE_URL`
- `useful_queries.sql` — **новый файл**, готовые SQL-запросы для просмотра данных


- **Сессии в памяти** (`sessionStore.js`) — при перезапуске бота все данные пользователей теряются. Для продакшена нужно подключить Redis или Postgres.
- **Голосовые сообщения** не поддерживаются — только текстовые ответы в интервью. Для голоса нужно добавить транскрибацию (например, Whisper API) в `bot.js` (`bot.on('voice', ...)`).
- **Генерация фото** — заглушка. Нужно подключить реальный image-API (OpenAI Images, Stability AI, Replicate) в `src/handlers/photo.js`, функция `generateImage()`.
- **Один пользователь = один активный процесс за раз** — если пользователь во время интервью пришлёт файл резюме, это может сбить состояние. Для продакшена нужна более строгая state machine.
- Не тестировался живьём (нет сети в среде разработки) — код синтаксически проверен, но перед продакшеном обязательно прогони ручной smoke-тест по всем командам.

## Экспорт в Google Таблицу

1. Создай Google Таблицу вручную, создай в ней 2 листа (вкладки внизу): **Users** и **ResumeAnalyses**
2. В Google Cloud Console создай сервисный аккаунт и включи Google Sheets API — подробная пошаговая инструкция прямо в комментариях файла `src/integrations/googleSheets.js`
3. Дай сервисному аккаунту доступ "Редактор" к своей таблице (через кнопку "Настройки доступа" в самой таблице, вставив email сервисного аккаунта)
4. В Railway Variables добавь `GOOGLE_SERVICE_ACCOUNT_EMAIL`, `GOOGLE_PRIVATE_KEY`, `GOOGLE_SHEET_ID`
5. После этого при `/start` в лист **Users** будет добавляться строка, а при `/analyze` — в лист **ResumeAnalyses**

## Экспорт в amoCRM

1. В amoCRM: Настройки → Интеграции → создай приватную интеграцию → вкладка "Ключи и доступы" → сгенерируй "Долгосрочный токен"
2. В Railway Variables добавь `AMOCRM_BASE_URL` (например `https://mycompany.amocrm.ru`) и `AMOCRM_ACCESS_TOKEN`
3. После этого при загрузке резюме пользователем бот будет автоматически создавать контакт + лид в amoCRM с примечанием
4. Подробности и нюансы (если в тарифе нет долгосрочных токенов) — см. комментарии в `src/integrations/amocrm.js`

## Монетизация: платные функции

По умолчанию **всё бесплатно** (удобно для теста). Чтобы сделать `/improve` и `/tailor` платными, а `/analyze` и `/interview` оставить бесплатными:

1. В Telegram напиши **@BotFather** → `/mypayments` → выбери своего бота
2. Подключи платёжного провайдера — для России обычно доступна **ЮKassa**. Следуй инструкциям BotFather, понадобится подключить свой аккаунт ЮKassa (или другого провайдера из списка)
3. BotFather выдаст **Payment provider token**
4. В Railway Variables добавь:
   ```
   PAYMENT_PROVIDER_TOKEN = токен_от_botfather
   PRICE_IMPROVE_RUB = 199
   PRICE_TAILOR_RUB = 299
   ```
5. С этого момента при `/improve` или `/tailor` бот сначала показывает счёт на оплату прямо в Telegram (нативный интерфейс оплаты картой), и только после успешной оплаты выполняет саму доработку резюме

**Как это устроено технически:** `/analyze` и `/interview` не изменились и остаются бесплатными в любом случае. `/improve` и `/tailor` проверяют, подключены ли платежи (`PAYMENT_PROVIDER_TOKEN` задан) — если да, выставляют счёт через `ctx.replyWithInvoice`; если платежи не настроены, функции по-прежнему работают бесплатно (удобно для разработки/теста без реальных денег).

Каждый платёж автоматически сохраняется в таблицу `payments` в базе данных (если БД подключена) — для учёта и сверки доходов.

## Виджет анализа резюме на сайте (Tilda) — лидогенерация

Бот теперь работает не только в Telegram, но и отдаёт HTTP API, который можно встроить в любой сайт, включая Tilda.

### Как это работает

1. Посетитель сайта заполняет имя/email/телефон, загружает резюме
2. Виджет отправляет запрос на твой Railway-сервер (`/api/analyze`)
3. Сервер анализирует резюме через Claude, показывает результат прямо на сайте
4. Одновременно лид сохраняется в Postgres, Google Sheets и создаётся в amoCRM — автоматически

### Шаг 1: Узнай публичный адрес своего Railway-проекта

1. Открой проект на Railway → вкладка **Settings** → раздел **Networking**
2. Если домена ещё нет — нажми **Generate Domain**
3. Скопируй адрес, будет выглядеть как `https://resume-bot-production.up.railway.app`

### Шаг 2: Проверь, что API отвечает

Открой в браузере `https://твой-адрес.up.railway.app/api/health` — должен вернуться `{"status":"ok"}`. Если нет — смотри логи Railway, вероятно порт не проброшен (Railway делает это автоматически при генерации домена).

### Шаг 3: Вставь виджет в Tilda

1. Открой файл `tilda-widget.html` из этого проекта
2. Замени строку `const API_URL = 'https://ТВОЙ-ПРОЕКТ.up.railway.app/api/analyze';` на свой реальный адрес из шага 1 (не забудь `/api/analyze` в конце)
3. В Tilda: добавь блок **T123 "HTML-код"** на нужную страницу
4. Вставь весь код из `tilda-widget.html` в этот блок
5. Опубликуй страницу

### Шаг 4 (рекомендуется): ограничь CORS своим доменом

По умолчанию API принимает запросы с любого сайта (`CORS_ORIGIN` не задан). Когда виджет заработает на Tilda, добавь в Railway Variables:
```
CORS_ORIGIN = https://твой-сайт-на-тильде.ru
```
Это защитит API от использования на чужих сайтах.

### Куда попадают лиды с сайта

- **Postgres**: таблица `web_leads` (можно смотреть через `SELECT * FROM web_leads ORDER BY created_at DESC`)
- **Google Sheets**: лист **WebLeads** (создай этот лист в таблице заранее, как и остальные)
- **amoCRM**: новый лид с названием "Сайт: анализ резюме — [имя/email]"
